Welcome to the Deliberately Vulnerable Web Application!

The aim of this workshop is to learn more about “Insecure Direct Object References” (IDORs). As the name suggests, IDOR vulnerabilities occur when web applications allow users to change parameter values that directly reference objects and gain access to restricted material.

Despite the fact that OWASP lists them as one of the top web application vulnerabilities, many training tools do not focus on IDORs specifically. This application hopes to educate developers on what IDORs are, how to find them, and how to mitigate against them.

Your goal is to complete each exercise by finding the flags and entering them in the box on the sidebar. Once you are done, please look through the fixed version of the application and see the changes made in the code to patch the vulnerabilities. Have fun!
